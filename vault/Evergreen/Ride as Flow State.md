# Ride as Flow State

A ride is a structured live experience — phases that carry attention like a current.

## Phases

`lobby → showcase → bidding → cooldown → archived`

Each phase has one job. Adaptive anti-snipe extends bidding when late energy arrives — the system flexes without breaking shape.

## Design lesson for the vault

Workflows should feel like rides: clear phases, one purpose each, room to adapt when real energy shows up.
