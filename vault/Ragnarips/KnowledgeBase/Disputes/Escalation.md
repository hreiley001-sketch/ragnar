# Dispute Escalation Ladder

**Rule:** Every dispute follows the same ladder — intake → evidence → decision → trust score update.

**Ladder:**
1. **Intake** — buyer opens dispute with a clear reason; order marked disputed.
2. **Triage** — Counsel classifies: shipping delay, condition mismatch, authenticity/fraud, other.
3. **Evidence (72h)** — seller and buyer submit tracking, photos, grading labels, chat screenshots.
4. **Decision** — staff resolve `resolved_refund` or `resolved_denied` with a written resolution.
5. **Trust update** — fraud score recomputed; repeated refund losses may trigger restrict/suspend playbooks.

**Agent action:** Cite this ladder; never promise refunds outside policy; always create an audit trail (SupportAuditLog / TrustEvent).
